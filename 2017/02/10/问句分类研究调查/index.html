    • 问句分类(Question Classification)概述

    K老板说既然要做”基于知识库的问答”(knowledage-based question answering)，那我们先把”问句分类”(Question Classification)撸一遍吧。

    Good Suggestion (尼玛，其实我是想用满屏的弹幕吐槽你啊!). 于是乎风风火火的搞起了文献调研.

    什么是问句分类？

    问句分类任务的目标在于，构建一个数学模型M，在给定一个自然语言问句Q和指定问句分类体系\(S = (S_1, S_2, …, S_n) \)的条件下，模型能够输出Q属于\(S_i \)的概率 \(P(S_i| Q) \).

    因此问句分类中最重要的两个问题是:

    1. 如何定义分类体系S?
    2. 用什么方法构建数学模型M?

    值得注意的是，多数问句分类研究是”依据答案估计问句类型”，即\(S_i\)往往与问句的答案相关.

    E.g., “Q:Who is Donald Trumlp?” -> “S_i:People”. 这里问句类型是People.

    问句分类体系

    问句分类往往并非作为单独的应用，而是服务于搜索、问答等其他面向用户的应用，因此好的分类体系必然对构建面向用户的应用产生影响.

    然而分类体系构建这样的技术活，的确不是我这种连词表构建都没玩转的信息管理系学渣能够彻底掌握的.

    当然啦，大部分看官们也不用担心，咱们自己不懂，借用大牛们的就是啦.

    目前，主要的问句分类体系有:

    • Qtargets[1]，包括122个类型答案类型;
    • BBN[5]，包括29个主类型和64个次级类型;
    • UIUC[11]，包括6个主类型和50个次级类型;
    • IBM[12]，包括2500种问句类型;(详细内容未公开)
    • MARCO[10]，包括5种问句类型，description/numeric/entity/person/location.

    问句分类方法

    问句分类研究中构建模型M的主要方法包括手工规则法[1][2]、概率模型法[3-5]、机器学习法[6-9].

    当前的主流方法是机器学习方法，即人工特征+监督学习:

    • 人工特征，句法特征(Words, Phraes, N-grams, POS Tags)和语义特征(Wh-Words, Head Words, Name Entities, Related Words, Question Patterns);
    • 监督学习，分类算法(CRF, SVM)和回归算法.

    据文献记载(实测也的确如此)，基于机器学习的方法在多个数据集上都能获得90%以上的正确率.
