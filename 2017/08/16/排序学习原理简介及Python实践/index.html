    嗯哼，以本汪的语文能力，实难寥寥数语解释透彻. 所以，各位看官请戳这里.

    如果你也是科研汪，并想深入了解LtR，那么请参考大神刘铁岩的这篇论文.

    相信大家看完KubiCode的博客，已经对LTR有了一个基本的了解. 正如KubiCode所述，一个完整的LTR实验框架包括了以下五个步骤:

    1.训练数据获取 -> 2.特征提取 -> 3.模型训练 -> 4.测试数据预测 -> 5.效果评估

    然而，在大牛们疯狂刷灌水的论文中，能玩的出花的两个步骤当然是特征提取和模型训练.

    • 特征提取，即将Query-Document对转化为向量表示. 根据任务的不同，可使用的特征真是多种多样.
    • 模型训练，即为LTR设计最优化目标函数. 主要有Pointwise，Pairwise和Listwise三种.
